> 최종 업데이트: 2026-03-06
> 기획서 v6 + 전략 논의 결과 통합

---

## 1. 프로젝트 개요

**뇌피셜**은 찰리 멍거의 25가지 인지 편향을 기반으로 한 AI 대화형 웹앱입니다.

- **앱 이름:** 뇌피셜
- **부제:** 내 뇌가 공식이라 우기는 것들
- **타겟:** 10대 후반~30대, 심리테스트/자기분석에 관심 있는 한국 사용자
- **핵심 철학:** "AI는 답을 주지 않는다. 더 좋은 질문을 준다."
- **톤:** 친근하고 가벼운 반말, 이모지 적절히, 학술 용어 사용 금지

### 제품 전략
- **Step 1 (편향 진단)** = 바이럴 훅 (결과 카드 공유 → 신규 유입)
- **Step 2 (고민 분석)** = 리텐션 엔진 (고민이 생길 때마다 재방문)

### 정책 메모
- **비로그인 결과는 서버 세션 기준:** 익명 사용자의 진행 상태와 결과는 서버 세션을 기준으로 복구됨. `localStorage`는 임시 캐시/복구 보조 용도
- **비로그인 Step 2 결과도 서버 저장:** 익명 사용자의 고민 분석 결과까지 서버 세션에 저장하고, 로그인 시 사용자 계정으로 마이그레이션
- **비로그인 사용 제한은 best-effort:** 익명 사용자의 "1회 무료"는 강한 보장이 아니라 체험용 제한. UI 문구도 과도하게 단정하지 않음
- **전체 리포트는 MVP 무료:** 단, CTA/레이아웃은 향후 유료 전환 가능성을 고려해 설계

## 4. 사용자 플로우 & 화면 목록

### 4-1. 전체 플로우

```
[첫 방문 — 비로그인]

  랜딩 페이지
      │
      ▼
  Step 1: AI 대화 (7~12턴)
      │
      ▼
  Step 1 결과 페이지
      │
      ├── 공유 카드 다운로드 / 카카오 공유
      │
      ▼
  Step 2 맛보기 유도 ("고민 하나, 무료로 분석해볼래요?")
      │
      ▼
  Step 2: 고민 입력 → 분석 결과 (무료 체험, 결과 저장 가능)
      │
      ▼
  로그인 유도 ("이 결과를 내 계정에 이어서 저장할래요?")


[재방문 — 로그인]

  로그인 → 대시보드
      │
      ├── 편향 프로필 요약
      ├── Step 2 분석 히스토리 (최근 10개)
      ├── "새 고민 분석하기" 버튼
      └── "편향 재검사" 버튼
```

| # | 화면 | 유형 | 핵심 요소 |
|---|------|------|-----------|
| 1 | **랜딩 페이지** | 정적 | 헤드라인 + CTA 버튼 |
| 2 | **Step 1 대화 UI** | 인터랙티브 | 채팅 버블 + 선택지 버튼 + 진행 바 |

### 5-2. Step 1 대화 UI

```
┌─────────────────────────────┐
│ 🧠 뇌피셜 분석 중... ████░░ 65% │
├─────────────────────────────┤
│                             │
│  🤖 요즘 뭔가 사고 싶은 거   │
│     있어요?                  │
│                             │
│  ┌──────────┐ ┌──────────┐  │
│  │ 전자기기  │ │ 옷/패션  │  │
│  └──────────┘ └──────────┘  │
│  ┌──────────┐ ┌──────────┐  │
│  │ 음식/맛집 │ │   없음   │  │
│  └──────────┘ └──────────┘  │
│                             │
└─────────────────────────────┘
```

**핵심 요소:**

1. **진행 바 (상단 고정)**
   - 확신도 기반 % 표시
   - "🧠 뇌피셜 분석 중... ████████░░ 78%"
   - 편향이 뚜렷한 사람: 7~8턴에 100%, 고른 사람: 10~12턴

2. **AI 메시지 버블**
   - 친근한 반말 톤
   - 이모지 적절히 사용

3. **터치 선택지 (90%)**
   - 4개 옵션 버튼 (2x2 그리드 or 세로 나열)
   - 탭하면 즉시 disabled (연타 방지)
   - 선택 후 다음 질문까지 로딩 애니메이션

4. **짧은 타이핑 입력 (10%, 전체 1~2회만)**
   - Auto-focus (입력창 나타나면 키보드 자동 올라옴)
   - 플레이스홀더: "예: 아이패드", "예: 퇴사"
   - 단어~문장 수준의 짧은 입력

5. **로딩 애니메이션 (매 턴)**
   - 단순 스피너 **절대 금지**
   - "🧠 뇌피셜 분석 중...", "찰리 멍거의 뇌와 동기화 중..." 등
   - 마이크로 애니메이션으로 "분석하고 있다"는 느낌 → 신뢰감
   - 체감 로딩 1초 이내 목표

6. **에러 상태**
   - 네트워크 끊김: "인터넷 연결을 확인해주세요" 안내
   - API 실패: "잠시 후 다시 시도해주세요" + 재시도 버튼

7. **진행률 표현 가이드**
   - 진행 바는 "정밀 진단 수치"가 아니라 사용자 안심용 진행 표현
   - `%` 숫자를 쓰더라도 실제 편향 점수를 직접 암시하지 않음
   - 문구 예시: "거의 다 왔어요", "Top 3를 정리 중이에요"

**핵심 요소:**

1. **Top 3 캐릭터 카드**
   - 순위 + 캐릭터명 + 부제 + 한 줄 설명
   - MBTI 결과 카드 느낌

2. **25개 레이더 차트**
   - Top 3~5: 강한 색상으로 강조 (레이블 굵게)
   - 나머지 20개: 흐리게 처리 (연한 색 + 얇은 선)
   - 전체 25개 라벨이 다 보이되, 시각적 계층으로 가독성 확보

3. **편향별 상세 해설**
   - 접기/펼치기 (아코디언)
   - 펼치면: 2~3문단 설명 + 발동 상황 4개 + 대표 인물

4. **리포트 CTA**
   - "전체 분석 리포트 받기" 버튼
   - MVP에서는 무료 다운로드
   - 버튼 구조와 레이아웃은 향후 유료 전환에도 그대로 재사용 가능하게 설계

5. **광고 슬롯**
   - **위치:** Top 3 해설 아래, Step 2 CTA 위
   - **크기:** 배너 320x100 (모바일) / 적응형 (데스크톱)
   - 콘텐츠와 자연스럽게 어우러지도록 여백 처리

6. **공유 버튼 그룹**
   - 이미지 다운로드 (기본)
   - 카카오톡 공유 (보조)
   - 링크 복사

7. **Step 2 유도 CTA**
   - "이 편향이 실제 고민에서 어떻게 작동하는지 볼래요?"
   - Step 2로 자연스러운 전환

8. **토스 후원**
   - 페이지 최하단에 작게 배치

9. **결과 복구 UX**
   - 새로고침/재진입 시 결과가 유지되는 경험을 기본으로 설계
   - 비로그인 결과도 "로컬 임시 저장본"이 아니라 서버 세션에서 복구되는 흐름 기준
   - Step 2 분석 결과도 익명 상태에서 서버 저장 후, 로그인 시 계정 히스토리로 이어지는 흐름 기준

**핵심 요소:**

1. **렌즈 카드** (3~4개 선별)
   - 이모지 + 렌즈명 + 판정(찬성/반대/조건부) + 근거 한 줄
   - 판정별 시각적 구분: ✅ 녹색, ❌ 빨간색, ⚠️ 노란색
   - 색만으로 상태를 구분하지 않고 아이콘 + 텍스트 배지 함께 사용

2. **나머지 렌즈**
   - "더 보기" 접기/펼치기
   - 제외 이유 간단히 표시

3. **편향 경고 섹션**
   - 강조 배경색 (경고 느낌)
   - Step 1 Top 편향이 이 고민에서 어떻게 작동하는지

4. **결정적 질문** (2~3개)
   - 번호 매기기
   - 구체적이고 답변 가능한 형태
   - **답은 제공하지 않음** — 사용자가 스스로 판단

5. **광고 슬롯**
   - **위치:** 결정적 질문 섹션 아래
   - **크기:** 배너 320x100

6. **사용 제한/로그인 유도 문구**
   - 비로그인 초과 상태는 단정형보다 완화된 문구 우선

## 12. 마스코트 가이드

### 캐릭터 정의

- 기본 캐릭터: **줄무늬 고양이 장인/메이커**
- 핵심 요소:
  - 큰 눈
  - 고글
  - 가죽 앞치마
  - 작은 공구 또는 작업자 분위기
- 역할:
  - 제품의 딱딱함을 낮추는 안내자
  - 로딩/빈 상태/공유 이미지에서 브랜드 기억점을 만드는 장치

### 스타일 방향

- **실사 복제보다 재해석 우선**
- 기본 원본 이미지를 그대로 사실적으로 반복 사용하지 않고, **픽셀아트 느낌으로 단순화한 버전**을 우선 사용
- 권장 톤:
  - 16-bit 또는 하이브리드 픽셀 일러스트
  - 또렷한 실루엣
  - 제한된 색상 수
  - 과도한 디테일보다 표정과 아이코닉함 우선
- 제품 메인 톤과 맞게 핑크/퍼플 포인트 조명이나 보조 글로우를 더할 수 있음

### 권장 사용 위치

- 랜딩 히어로 보조 비주얼
- Step 1/Step 2 로딩 상태
- 대시보드 빈 상태
- 공유 카드의 스티커/보조 엠블럼
- 온보딩성 설명 카드의 장식 일러스트

### 과사용 금지 위치

- Step 1 결과 Top 3 카드 메인 비주얼 대체 용도
- Step 2 렌즈 카드 내부 핵심 정보 영역
- 레이더 차트, 데이터 시각화 위 직접 중첩
- 긴 문단 설명 옆의 반복 장식

### 사용 원칙

- 마스코트는 **정보 전달의 주인공이 아니라 브랜드 안내자**
- 결과 화면에서는 정보 카드보다 더 튀지 않게 배치
- 한 화면에 1회 등장 정도를 기본으로 함
- 빈 상태/로딩 상태에서는 상대적으로 존재감을 더 줘도 됨

### 구현 메모

- 정적 PNG 외에 SVG 또는 스프라이트 느낌의 자산도 고려
- 픽셀아트 버전은 작은 썸네일에서도 알아보이게 대비를 높임
- transparent 배경 버전 + 단색 배경 버전 둘 다 준비하면 활용 범위가 넓음

### 화면별 추천 위치

- 히어로 비주얼 옆 또는 아래의 보조 캐릭터
- CTA 아래의 작은 스티커형 장식
- "어떻게 성찰하나요?" 섹션 설명 카드의 코너 장식

권장:
- 메인 헤드라인을 방해하지 않는 중간 크기
- 인지 지도/노드 그래픽과 같이 배치

비권장:
- 히어로 중앙의 유일한 메인 오브젝트로 단독 사용
