# 뇌피셜 운영 비용 가이드

## 현재 운영 기본값

- 비로그인 사용자
  - Step 1: 하루 1회
  - Step 2: 하루 1회
- 로그인 사용자
  - Step 1: 하루 1회
  - Step 2: 하루 3회
- Step 1 질문 생성
  - 기본값은 하이브리드 모드
  - Gemini 호출 턴: `0,3,6`
  - 나머지 턴은 서버 질문 은행 사용
- 기본 모델
  - Step 1: `gemini-2.5-flash-lite`
  - Step 2: `gemini-2.5-flash-lite`

## 코드에 반영된 비용 절감 장치

- Step 1 결과는 한 번 생성하면 캐시를 재사용한다.
- Step 2 결과는 같은 고민 텍스트 기준으로 캐시를 재사용한다.
- 결과 화면, 대시보드, 공유 카드에서 같은 결과를 다시 열 때 모델을 다시 부르지 않는다.
- Gemini 호출 수는 일별로 `dailyGeminiMetrics/{YYYY-MM-DD}` 에 기록된다.
- 사용자별 일일 사용량은 `dailyUsage` 컬렉션에 기록된다.

## Firestore에서 확인할 컬렉션

- `dailyUsage`
  - 사용자 또는 익명 scope별 일일 사용량
  - 주요 필드
    - `step1_starts`
    - `step2_runs`
- `dailyGeminiMetrics`
  - 일자별 Gemini 총 호출 수
  - 주요 필드
    - `total_calls`
    - `models.*`
    - `sources.*`

## 운영자가 꼭 해야 하는 일

앱 코드만으로는 Google Cloud Billing 예산 알림을 만들 수 없다. 아래 작업은 콘솔에서 직접 해야 한다.

1. Google Cloud Console 접속
2. `Billing`
3. 현재 Firebase/Gemini가 연결된 결제 계정 선택
4. `Budgets & alerts`
5. 새 예산 생성

권장 시작값:

- 테스트 단계
  - 월 예산: 매우 낮게 시작
  - 알림: 25%, 50%, 75%, 90%, 100%
- 초기 공개 단계
  - 월 예산: 감당 가능한 소액 한도
  - 알림: 50%, 75%, 90%, 100%

## 운영 체크리스트

- 공개 전
  - Firebase Auth 로그인 정상 동작 확인
  - 비로그인 Step 1 두 번째 시도 차단 확인
  - 비로그인 Step 2 두 번째 시도 차단 확인
  - 로그인 Step 2 네 번째 시도 차단 확인
  - `dailyGeminiMetrics` 증가 확인
- 공개 후
  - 하루 1회 이상 `dailyGeminiMetrics.total_calls` 확인
  - `dailyUsage`에서 비정상적으로 큰 익명 트래픽이 있는지 확인
  - 예산 알림 메일 수신 여부 확인

상세 QA 절차는 [`뇌피셜_QA_테스트_시나리오.md`](C:\projects\3-brain-official\docs\뇌피셜_QA_테스트_시나리오.md) 참고.

## 비용이 다시 올라가기 시작하면

1. Step 2 모델만 먼저 낮춘다.
2. Step 1의 AI 생성 턴 수를 더 줄인다.
   - 예: `STEP1_AI_TURNS=0,4`
3. Step 2 비로그인 체험을 더 강하게 제한한다.
4. 필요하면 Step 2만 로그인 전용으로 바꾼다.
