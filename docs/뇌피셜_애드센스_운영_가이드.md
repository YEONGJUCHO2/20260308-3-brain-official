# 뇌피셜 애드센스 운영 가이드

## 1. 현재 결론

뇌피셜은 애드센스를 붙일 수 있는 주제다. 다만 승인 전에 아래 조건을 먼저 만족시키는 편이 안전하다.

- 깨진 한글, 미완성 문구, 빈 상태를 줄인다.
- 개인정보처리방침과 이용약관을 실제 운영 문구로 보강한다.
- 광고를 CTA, 공유 버튼, 로그인 버튼 근처에 붙이지 않는다.
- 한 화면에 광고보다 콘텐츠가 더 많아야 한다.

## 2. 이번 버전 광고 위치

이번 버전은 3곳만 사용한다.

1. Step 1 결과
   - 파일: [step-one-result-page.tsx](C:\projects\3-brain-official\src\components\app\step-one-result-page.tsx)
   - 위치: 전체 리포트 CTA 아래, 공유 섹션 위

2. Step 2 결과
   - 파일: [step-two-result-page.tsx](C:\projects\3-brain-official\src\components\app\step-two-result-page.tsx)
   - 위치: 제외된 렌즈 섹션 아래, 다음 행동 CTA 위

3. 대시보드
   - 파일: [dashboard-page.tsx](C:\projects\3-brain-official\src\components\app\dashboard-page.tsx)
   - 위치: 최근 기록 아래, 다음 고민 카드 위

랜딩 첫 화면에는 넣지 않는다. 승인 전에는 특히 첫 화면 광고를 피하는 편이 안전하다.

## 3. 구현 구조

- 공통 스크립트: [adsense-script.tsx](C:\projects\3-brain-official\src\components\ads\adsense-script.tsx)
- 공통 슬롯: [ad-slot.tsx](C:\projects\3-brain-official\src\components\ads\ad-slot.tsx)
- 전역 스타일: [globals.css](C:\projects\3-brain-official\src\app\globals.css)

환경변수가 비어 있으면 실제 광고 대신 자리표시자만 보인다.

## 4. 환경변수

[.env.local.example](C:\projects\3-brain-official\.env.local.example) 기준:

```env
NEXT_PUBLIC_ADSENSE_CLIENT=ca-pub-your-client-id
NEXT_PUBLIC_ADSENSE_TEST_MODE=on
NEXT_PUBLIC_ADSENSE_SLOT_STEP1_RESULT=1234567890
NEXT_PUBLIC_ADSENSE_SLOT_STEP2_RESULT=2345678901
NEXT_PUBLIC_ADSENSE_SLOT_DASHBOARD=3456789012
```

운영 전환 시:

- `NEXT_PUBLIC_ADSENSE_CLIENT` 에 실제 `ca-pub-...` 입력
- 슬롯 ID 3개 입력
- 승인 전/테스트 중에는 `NEXT_PUBLIC_ADSENSE_TEST_MODE=on`
- 승인 후 운영에서는 `off` 또는 제거

## 5. 절대 피할 배치

- 랜딩 첫 화면 상단
- Step 1 Top 1 카드 내부
- Step 2 생각 경고등 카드 내부
- 공유 버튼과 너무 가까운 위치
- 로그인 버튼 바로 아래
- 버튼처럼 보이는 광고 스타일

## 6. 승인 전 체크리스트

- 개인정보처리방침 문구 실제 운영 기준으로 보강
- 이용약관 문구 실제 운영 기준으로 보강
- 깨진 한글 정리
- 빈 화면/에러 화면 점검
- 광고가 없어도 페이지가 충분히 유용한지 확인
- 모바일에서 광고와 CTA가 오인되지 않는지 확인

## 7. 추가 메모

- EEA/UK 대상이면 CMP/동의 배너까지 검토해야 한다.
- 애드센스 승인 후에도 처음엔 슬롯 수를 늘리지 말고, 현재 3개 위치만 유지하는 편이 안전하다.
